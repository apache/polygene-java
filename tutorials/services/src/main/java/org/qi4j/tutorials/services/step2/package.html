Step 2 - Activatable

Services can be "activated" and "passivated". Applicaitons can be notified of this occurring by Qi4j runtime calling the two methods activate and passivate of the Activatable interface implemented by any number of Mixin implementations. That means that unlike other methods in a composite all mixins implementing the method will be called.

To showcase how this works, we refactor the code to create a number of copies of the books, to be lend out upon call to the borrowBook method, which will return null if no copy is available. The book copies are created in the activate method.

Steps to do.

  1. Make LibraryMixin implement the org.qi4j.api.service.Activatable interface.
  2. In the activate method, create a couple of books of each title and store each copy in a HashMap<String,ArrayList<Book>>.
